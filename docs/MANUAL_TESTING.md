# Ручная проверка изменений — начиная с удаления почты

Дата составления: 2026-05-25. Покрывает все коммиты от `7a763ee` (удаление
раздела Почта) и далее, включая последнюю сессию (умный подбор + иконки).

## 0. Запуск стенда

**Предусловия:**
- Запущен PostgreSQL с базой `nirdb` (см. `application.yaml`).
- Установлен Java 17 и Node 18+.

**Порядок:**

```bash
# Терминал 1 — backend (порт 8080)
./gradlew bootRun

# Терминал 2 — frontend (порт 4200, проксирует на бэкенд)
cd frontend && npx ng serve --open
```

Дождаться в первом терминале строки `Started Nir2Application`, во втором —
`Local:   http://localhost:4200/`. Если автоматически не открылся браузер,
открыть `http://localhost:4200`.

**Учётные данные** (создаются `DataInitializer` при первом старте):

| Логин | Пароль | Роль |
|---|---|---|
| `admin` | `admin` | ROLE_ADMIN — видит всё + редактирование |
| `operator` | `operator` | ROLE_OPERATOR — только просмотр + работа с заявками |

Залогинься как `admin` для полной проверки.

## 7. **СППР-умный подбор оборудования (НОВОЕ)** — коммиты `e4d8b02` + `b7f8e72`

⭐ **Главная фича этой итерации.** Подбор оборудования теперь не плоский
список по алфавиту, а scoring 0–100 по 4 критериям с пользовательскими
весами и объяснимостью.

### 7.1 Открыть подбор

- [ ] Сайдбар → **Тендеры** → открыть любой активный тендер (например
      `0373100065021000041` — там УЗИ + Монитор лоты).
- [ ] В таблице лотов у каждой строки есть кнопка «Подобрать».
- [ ] Клик «Подобрать» — под таблицей лотов разворачивается панель
      «Подбор оборудования для лота №N».

### 7.2 Тулбар пресетов

- [ ] 4 кнопки: `Баланс`, `Макс. прибыль`, `Надёжность`, `Свой`.
- [ ] Активный пресет подсвечен синим (`Баланс` по умолчанию).
- [ ] Переключение пресета — таблица мгновенно перестраивается
      (отправляется POST `/api/equipment/match/{lotId}` с новыми весами).

### 7.3 Кастомные веса (CUSTOM)

- [ ] Выбрать «Свой» — появляется панель из 4 слайдеров: `Цена`, `Маржа`,
      `Опыт`, `Габар.` (0–100 каждый).
- [ ] Под слайдерами счётчик `Σ = ...` — сумма всех весов
      (нормализуется на сервере к 1.0).

- [ ] Перезагрузить страницу → выбранный пресет и значения слайдеров

### 7.4 Cold-start банер

Если в базе нет WON-заявок:

- [ ] Над таблицей жёлтый банер `⚠ Истории сделок пока нет — рекомендации
      основаны только на габаритах.`

### 7.5 Рекомендация СППР

- [ ] Если у топ-1 кандидата `score ≥ 60` — над таблицей синяя плашка
      `⭐ Рекомендация СППР: <название> (N баллов)`.
- [ ] Если есть лучший дистрибьютор — приписка
      `· лучший дистрибьютор: <имя> (ср. маржа X%)`.
- [ ] Сама строка top-1 в таблице подсвечена голубым.
- [ ] Если у top-1 score < 60 (например, cold-start) — плашки нет, строки

8 колонок: `#`, `Наименование`, `Score`, `Цена`, `Маржа`, `Опыт`, `Габар.`, `[+]`.

- [ ] В колонках Цена/Маржа/Опыт/Габар. — мини-прогрессбары (зелёный

- [ ] Если по какому-то критерию нет данных (cold start) — прогрессбар

### 7.7 Раскрытие строки

### 7.9 Smoke-тест бэкенда

В отдельном терминале:

```bash
# Подбор с пресетом Баланс
curl -s -X POST http://localhost:8080/api/equipment/match/1 \
  -H 'Content-Type: application/json' \
  -d '{"preset":"BALANCED"}' | python3 -m json.tool | head -40
```

- [ ] Возвращается JSON со структурой
      `{lotId, hasHistory, weightsUsed:{price,margin,track,dim}, candidates:[...]}`.
- [ ] Каждый candidate содержит `score`, `rank`, `breakdown` с 4 sub-scores,
      `bestDistributor` (или null), `estimatedPrice`, `estimatedMargin`.

```bash
# Кастомные веса
curl -s -X POST http://localhost:8080/api/equipment/match/1 \
  -H 'Content-Type: application/json' \
  -d '{"preset":"CUSTOM","weights":{"price":50,"margin":50,"track":0,"dim":0}}' \
  | python3 -m json.tool | head -20

# Без body — берёт BALANCED по умолчанию
curl -s -X POST http://localhost:8080/api/equipment/match/1 | python3 -m json.tool | head -10

# GET-алиас (для обратной совместимости)
curl -s http://localhost:8080/api/equipment/match/1 | python3 -m json.tool | head -10
```

- [ ] Все 4 формы возвращают один и тот же контракт `EquipmentMatchResponse`.
- [ ] `CUSTOM` без `weights` или со всеми нулями возвращает 400.

```bash
# Нет такого лота
curl -s -o /dev/null -w '%{http_code}\n' \
  -X POST http://localhost:8080/api/equipment/match/99999
```

- [ ] Возвращает `404`.
