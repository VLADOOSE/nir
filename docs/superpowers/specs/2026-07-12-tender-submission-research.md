# Участие в электронных торгах (подача заявки) — исследование для следующего блока

**Дата:** 2026-07-12
**Статус:** research / pre-brainstorm. НЕ реализовано. Заключительный шаг воронки West-Med/Регион-Мед.
**⚠ Уровень верификации:** собрано из открытых источников порталов + доменных знаний. **Процессные детали (обеспечение, демпинг, формат вложений) подтвердить у владельца бизнеса (отца) до дизайна.** Не строить вслепую.

## 1. Где мы и зачем это

Воронка доведена до: тендеры обоих KZ-каналов импортируются → ТЗ лотов разбирается (goszakup + СК-Фармация) → реестр/каталог-подбор → запрос КП поставщикам → сбор цен → **выбор победителя по лоту**. **Следующий большой шаг — сформировать и подать заявку на участие в тендере** (после сбора цен), т.к. West-Med/Регион-Мед сами выступают поставщиком на площадке.

Два канала (оба импортируются, см. CLAUDE §8):
1. **goszakup.gov.kz** — общий портал госзакупок РК. Способы: конкурс, запрос ценовых предложений (ЗЦП/ЦП), аукцион.
2. **СК-Фармация — `fms.ecc.kz`** (оператор АО «ЦЭФ») — единый дистрибьютор ГОБМП/ОСМС, **двухэтапный тендер** по медтехнике/медизделиям, ЭЦП.

## 2. Пакет заявки поставщика (что подаётся)

**goszakup, конкурс (по медтехнике):**
- **Техническая спецификация / форма соответствия** — построчно «требование заказчика → наш предлагаемый параметр» по каждому лоту. (У нас уже есть разобранное ТЗ заказчика + предложенная модель + характеристики — основа для авто-заполнения.)
- **Ценовое предложение** — цена за единицу/лот (у нас = `PriceRequest.responsePrice` / `ApplyItem.offeredCost` победителя + наценка).
- **Обеспечение заявки** — как правило **1% от суммы** (деньги на счёт площадки / банковская гарантия). *(Точный процент и способ — подтвердить.)*
- **Квалификационные требования** — правоспособность, отсутствие налоговой задолженности (> 6-кратного МРП), отсутствие в РНУ (реестр недобросовестных). **Портал проверяет их АВТОМАТически** по БИН из гос. баз → нам генерировать/прикладывать не нужно.
- **Сведения о субъекте / бенефициаре, согласие с условиями.**
- Всё подписывается **ЭЦП** (NCALayer / криптосокет на машине оператора).

**goszakup, ЗЦП (запрос ценовых предложений):** проще — **одно ценовое предложение** (изменить нельзя; до срока можно отозвать и переподать). Обеспечение обычно не требуется. *(Подтвердить.)*

**СК-Фармация (fms.ecc.kz), двухэтапный тендер:** 1-й этап — техническое предложение (без цен, на соответствие ТЗ), 2-й этап — ценовое. Документы требований видны на вкладке `?tab=documents` (мы её уже парсим для ТЗ: строки «Техническая спецификация», «Гарантийное обеспечение», «Перечень медтехники» и т.д. — это чек-лист того, что поставщик обязан приложить). ЭЦП обязательна.

## 3. Что АИС уже держит → может предзаполнить

- **Цену** — `PriceRequestItem.responsePrice` / выбранный победитель `ApplyItem.offeredCost` (+ единый контрол наценки).
- **Модель/бренд** — `TenderLot.proposedEquipment` (`MedEquipment`).
- **Регистрацию НЦЭЛС РК** — `MedEquipment.registration` (№ РУ, срок) — ключевой допуск для медтехники в РК.
- **Разобранное ТЗ заказчика** — `TenderLot.requiredSpec` + `max*` (goszakup И СК-Ф).
- → **АИС реально может собрать «форму соответствия» (требование → наш параметр → № РУ → цена) как черновик пакета.**

## 4. Граница автоматизации (реалистичная)

- **Автоматизируемо:** сборка/предзаполнение пакета (форма соответствия по ТЗ, ценовое предложение, подстановка № РУ, спецификация модели) → выгрузка в человекочитаемый/загружаемый вид (PDF/форма).
- **Ручное (у оператора):** **подпись ЭЦП + фактическая отправка на портал.** Полная программная подача = тяжёлая интеграция с криптосокетом NCALayer + необъявленными формами портала; риск/цена высоки. Реалистичный MVP: **«АИС готовит пакет → оператор подписывает ЭЦП и подаёт вручную»**, не сквозная автоподача.

## 5. Открытые вопросы к владельцу (отцу) — до дизайна

2. № РУ НЦЭЛС прикладывается **отдельным вложением** или встраивается в форму соответствия?
3. **Демпинг-порог** (мин. цена без спец-обоснования) — есть/какой?
4. **Обеспечение заявки** для ЗЦП — требуется? Процент/форма для конкурса?
5. Нужна ли сквозная **автоподача через ЭЦП**, или достаточно «АИС готовит пакет → подпись/подача вручную»?
6. Формат/шаблон формы соответствия (у заказчика бывает свой обязательный).

## 6. Предлагаемый первый шаг (когда дадут добро)

MVP «Генератор пакета заявки»: по тендеру с выбранными победителями → собрать **форму соответствия** (лот → требование ТЗ → предложенная модель/параметр → № РУ → цена с наценкой) + ценовое предложение → экспорт PDF/форма для ручной подписи и подачи. Схему подачи (goszakup vs fms.ecc.kz) — уточнить у отца перед дизайном.
