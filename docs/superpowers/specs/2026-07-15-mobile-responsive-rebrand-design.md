# Дизайн: мобильная адаптация + ребрендинг «АИС Медзакупки» + UI-ревизия

**Дата:** 2026-07-15 · **Статус:** согласовано (brainstorm) → план → SDD

## Цель
Сделать веб-АИС удобной с телефона (сейчас вёрстка десктоп-only: фикс-сайдбар 240px, таблицы без адаптива), переименовать в нейтральное **«АИС Медзакупки»** (сейчас логин хардкодит «АИС Регион-Мед» — путает KZ-пользователей), и попутно почистить актуальность/UX. Десктоп-вид НЕ меняется (всё под `@media (max-width: 900px)`).

## Решения (согласованы)
- **Название:** «АИС Медзакупки», подзаголовок «Тендеры и заявки на медоборудование». Логотип — медкрест (как PWA-иконка). Рынок — только в селекторе шапки после входа.
- **Брейкпоинт:** `900px`. Ниже — «мобильный» режим.
- **Поиск на мобиле:** переезжает наверх выезжающей панели (в узкой шапке ему тесно).
- **Объём:** максимум — включая карточки вместо таблиц на мобиле.

## 1. Нейминг / бренд
- `services/market.service.ts`: `companyLabel` больше НЕ несёт «АИС …» (шапка покажет продукт «АИС Медзакупки», а рынок — селектором). Оставить `symbol`/`logo`/market-метки для селектора; компанию для «О системе» отдавать по рынку.
- `pages/login/login.component.ts`: заголовок → «АИС Медзакупки» + подзаголовок; **убрать** строку «Тестовые данные: admin/admin»; логотип-монограмма «РМ» → медкрест-иконка.
- `layout/layout.component.ts`: слева шапки — «АИС Медзакупки» + крест (константа, не рыночная).
- `index.html`: `<title>` → «АИС Медзакупки» (уже частично сделано `apple-mobile-web-app-title=АИС`).
- `pages/about/about.component.ts`: описание — по рынку (компания из `market.service`), не хардкод «ООО Регион-Мед».

## 2. Мобильный макет (`layout.component.ts`)
- **Сайдбар → off-canvas drawer.** Сигнал `sidebarOpen` (default false). ≤900px: сайдбар `position: fixed; transform: translateX(-100%)`, при `sidebarOpen` → `translateX(0)` + затемняющий backdrop. Клик по ссылке ИЛИ по backdrop ИЛИ смена роута → `sidebarOpen=false`. >900px: сайдбар статичен как сейчас (drawer-стили не применяются).
- **Шапка ≤900px:** `[☰] АИС Медзакупки · [рынок] · [юзер]`. Кнопка ☰ только на мобиле (`display:none` на десктопе). Глобальный поиск — блоком наверху drawer (в шапке скрыт ≤900px).
- **Контент:** padding `24px 32px` → `12px 16px` ≤900px.

## 3. Таблицы → карточки (кросс-сквозной приём)
- В глобальном `styles.scss` — утилита `.responsive-cards` (opt-in класс на `<table>`): ≤900px `thead{display:none}`, `tr{display:block; border+radius+margin}` (карточка), `td{display:flex; justify-content:space-between}` с `td::before{content: attr(data-label)}` (подпись слева, значение справа). Глобальный стиль (не scoped) достаёт DOM компонентов.
- **Разметка по страницам:** на `<table>` добавить `class="responsive-cards"`, каждой `<td>` — `data-label="<Колонка>"`. Затрагиваемые списки: `tenders` (верхний список тендеров), `applies`, `equipment`, `facilities`, `distributors`, `private-requests`, `inbound`, `users`, `equipment-types`, `registry-reconciliation`, `dashboard` (таблица последних заявок).
- **Исключение — горизонтальный скролл, НЕ карточки:** плотные вложенные/матричные таблицы внутри карточки тендера (лоты, «Запросы КП», offer-comparison, ввод цен), smart-match, модалка `equipment-detail-modal`. Обернуть в `.table-scroll{overflow-x:auto; -webkit-overflow-scrolling:touch}`.

## 4. Тач-таргеты + чистка
- Глобально ≤900px: `button, .btn, a.btn { min-height: 40px }` (сейчас ~24px). Инпуты — `font-size:16px` на мобиле (iOS не зумит при фокусе).
- «Демо» единообразно (сейчас «Демо» в списке / «Контрольный пример» в карточке тендера — оставить «Демо»).
- Пустые состояния дашборда: добавить CTA-ссылки («Создать заявку» и т.п.).
- Бейджи ролей (admin/operator) — единый стиль (сейчас один solid, другой полупрозрачный).

## 5. Подход и изоляция
- **Кросс-сквозное** (drawer-механика — в `layout.component`; карточки-таблицы, тач-таргеты, padding — в глобальном `styles.scss`, один источник). Компонентные inline-стили НЕ дублируют — глобальные правила не scoped и достают их DOM по opt-in классам.
- **Десктоп без регресса:** все правила под `@media (max-width: 900px)`; вне медиа-запроса ничего не меняется.
- **Проверка:** Playwright на 390px — логин, дашборд, список тендеров, карточка тендера, заявки, дистрибьюторы; + `npm run build` (бюджет стилей) + десктоп-смоук что не разъехалось.

## Вне рамок (YAGNI сейчас)
- Карточки-рефлоу для матричных таблиц (offer-comparison/ввод цен) — остаются скроллом.
- Service worker / офлайн (PWA-обёртка уже есть без него).
- Тёмная тема, i18n, переработка навигации (nested-роуты).

## Разбивка задач (для плана/SDD)
1. Глобальный `styles.scss`: брейкпоинт-переменная, `.responsive-cards`, `.table-scroll`, тач-таргеты, инпут-16px, padding-хелперы.
2. `layout.component.ts`: drawer + ☰ + шапка-мобайл + поиск-в-drawer.
3. Ребрендинг: `market.service`, `login`, `index.html`, `about`, шапка.
4. Per-page (параллельно субагентами): проставить `responsive-cards`+`data-label` / `.table-scroll` по списку из §3; тач-таргеты где локально.
5. Чистка: тест-креды, «Демо», empty-state CTA, бейджи ролей.
6. Верификация Playwright@390 + build + десктоп-смоук.
